Архитектурные решения

1. Использование Vue 3 + Composition API
   Vue 3 выбран из-за простоты разработки и удобной реактивной модели. Composition API позволяет лучше структурировать логику компонентов.

2. Использование Express для backend
   Express выбран как простой и легковесный web-фреймворк для Node.js. Он подходит для небольших API сервисов.

3. Prisma ORM
   Prisma используется для:

- удобной работы с базой
- миграций
- генерации типов
  Также Prisma упрощает работу с SQLite.

4. SQLite как база данных
   SQLite выбран потому что:

- не требует отдельного сервера
- легко запускается локально
- подходит для тестового задания

5. Простая авторизация
   Для упрощения тестового задания используется выбор пользователя из списка. Полноценная авторизация (JWT, OAuth) не реализована.

6. Защита от race condition
   При взятии заявки в работу используется атомарное обновление:
   Если запись уже изменилась, возвращается `409 Conflict`.

7. Разделение frontend и backend
   Frontend и backend запущены как отдельные сервисы:

- frontend → Vite
- backend → Express API

Это стандартная архитектура SPA + API.
